Where I started
From the joined incident-level dataset (incidents + aggregated causes + aggregated consequences, one row per incident).
What I did
Strategy decision on the text corpus. I initially planned to merge the three text fields (incident description, causes, consequences) into a single corpus for clustering. After looking at the length distributions, I changed direction: incident descriptions average ~900 characters while causes and consequences are typically one short sentence each. Merging them would have buried the cause/consequence signal in the longer incident text. Decision: cluster on incident description only, and keep the causes/consequences as structured metadata for cluster validation later (similar to how the reference paper uses Basel event types).
HTML stripping. Incident descriptions contain HTML tags from the source system. Built a cleaning step using BeautifulSoup, then had to add a fix: some descriptions use <<...>> as quotation markers, which the parser was wrongly interpreting as HTML tags and stripping. I added a pre-processing step to neutralize these false-positive patterns before parsing.
Language detection. Discovered the corpus is multilingual: ~49% English, ~44% French, ~7% Italian, plus a negligible tail. This rules out an English-only approach (we'd lose half the data) and forces us to translate everything to English so we can cluster on a unified semantic space.
Current bottleneck — translation
Translation is where I'm stuck. Two options were on the table:

External APIs (DeepL, Google Translate): highest quality and easiest to set up, but sending incident descriptions to external services isn't acceptable from a data governance standpoint in a bank context.
Local model (Helsinki-NLP MarianMT): runs entirely on the Dataiku server, no external calls — the right choice for us. However, the standard install requires downloading the models from Hugging Face, and outbound access to Hugging Face is blocked from our environment.

I'm currently working around this by downloading the three models (FR→EN, IT→EN, ES→EN) on a machine with internet access, then uploading them as a Dataiku managed folder for the recipe to load offline. Setup is in progress.
